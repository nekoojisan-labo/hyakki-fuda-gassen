# 画像パーツと座標の直し方

このゲームでは、背景へカードやエフェクトを描き込まない。すべてを独立パーツとして、名前付きの枠へ配置する。

## パーツの置き場所

| パーツ | フォルダ | 変更時の注意 |
|---|---|---|
| 神社の背景 | `assets/backgrounds/` | カード枠や文字を描き込まない |
| 20種類のカード絵 | `assets/cards/` | カードIDと同じファイル名にする |
| カード枠・カード裏 | `assets/ui/` | 全カードで共通利用する |
| 斬撃・衝撃など | `assets/effects/` | 対象カードの中心へ追従させる |

カード画像は `src/data/cards.json` の `art` で対応付ける。画像を増やしたら `npm run validate` で欠落やID違いを検出できる。

## 座標の正本

盤面座標の正本は `src/data/layout.json` だけである。

- `desktop`: PC横画面。
- `mobile`: スマートフォン縦画面。
- `x`, `y`: 盤面左上を0、右下を100とした中心座標。
- `card.w`, `card.h`: 盤面に対するカード枠の幅と高さ。
- `player-monster-1` のような名前が実際の配置先になる。

CSSやゲーム処理へ個別座標を書かない。カードはスロット名へ入り、画面サイズに応じて実ピクセルへ変換される。

## コードを読まずに確認する方法

1. `npm start` で起動する。
2. 画面右上の「座標」を押す。
3. 12個の水色枠が重ならず、すべて画面内にあることを確認する。
4. PCとスマートフォンの両方で確認する。

自動確認は `npm run check`。PC 1440×900、スマホ390×844の実ブラウザ確認画像は `docs/qa-desktop.png` と `docs/qa-mobile.png` に保存する。
