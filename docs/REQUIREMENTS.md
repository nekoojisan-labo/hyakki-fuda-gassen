## 1. 目的・成功基準

- ブラウザ上で、プレイヤーが固定デッキを使いCPUとカードを出し合って戦える。
- 1試合を開始から勝敗決定まで5〜10分程度で遊べる。
- 遊戯王系の手触りを参考にしつつ、名称・世界観・カード・画像・UIはすべてオリジナルにする。
- 初版の利用者は制作者本人。面白さの核を検証する試作版とする。

## 2. 利用者と場面

- 利用者: 制作者本人。
- 場面: PCまたはスマートフォンのブラウザでローカル起動して遊ぶ。
- 保存先: `~/projects/browser-card-game`。

## 3. 入出力の実例（承認済み）

| 場面 | プレイヤーの入力 | ゲームの出力 |
|---|---|---|
| 通常召喚 | 星4・攻撃力1600の「鎌鼬」を攻撃表示で召喚 | 召喚権を1回消費し、そのターンに追加の通常召喚はできない |
| 上級召喚 | 「提灯小僧」1体を素材に、星6・攻撃力2300の「大天狗」を召喚 | 提灯小僧は墓地へ移動し、大天狗が場に出る |
| 境界例 | 攻撃力1800で、守備力2000の裏側守備モンスターへ攻撃 | 相手を表向きにする。攻撃側は破壊されず、攻撃側プレイヤーのライフが差分の200減る |

## 4. データ

### カードとデッキ

- 全20種類: モンスター12種、魔法5種、罠3種。
- プレイヤー用・CPU用の固定デッキを各20枚用意する。同じカードを複数枚採用できる。
- カード定義とデッキ定義は、対戦処理から分離したローカルJSONで管理する。
- カードには一意なID、名称、種類、星、攻撃力、守備力、効果、画像参照を持たせる。

### 画像・演出資産

- 背景、盤面装飾、カード絵、カード枠、カード裏、アイコン、エフェクトを独立ファイルにする。
- カード画像へ盤面背景やエフェクトを焼き込まない。
- 画像参照はカードID・エフェクトIDから解決し、CSSや対戦処理へ個別パスを散在させない。
- 原画は承認済みカンプの暗い和風幻想、墨色・深紅・古金・和紙・漆の方向性に合わせる。

### 座標・レイアウト

- 盤面要素を任意のピクセル座標へ直接置かず、意味のあるスロットIDへ配置する。
- 必須スロット: 双方の `monster-1..3`、`spell-trap-1..3`、手札、山札、墓地、プレイヤー情報、ターン終了、フェーズ表示。
- PC横画面とスマホ縦画面は別のレイアウト定義を持つ。
- 各スロットは論理座標の中心点、幅、高さ、基準点、重なり順を1つの定義ファイルで管理する。
- 実座標は表示領域とセーフエリアから算出する。コード内への座標マジックナンバー散在を禁止する。
- エフェクトはスロットIDまたはカードIDへ追従させ、対象の中心・辺・全体のいずれを基準にするか指定する。
- 実装時に座標・スロット名・当たり判定・基準点を可視化できるデバッグ表示を用意する。
- 異なる画面比率で、カードやエフェクトが指定スロットから外れないことを自動確認する。

## 5. 例外・エッジケースの決定

- 素材不足、召喚済み、空き枠なし、攻撃済み、発動条件未達などの不可能な操作は、実行ボタンを無効化する。
- 不可能な理由を「素材が1体必要」など短文で表示する。
- ページ再読み込みやブラウザ終了後は、直前のターン開始時点から再開する。
- 保存データが不正・破損している場合は、そのデータを使わず新しい試合を開始できる案内を出す。
- 山札がなくドローできないプレイヤーは敗北とする。
- 盤面3枠が埋まっている場合、新しいモンスターは召喚できない。

## 6. 非機能

- 対応画面: PC横画面およびスマートフォン縦画面。
- 対応ブラウザ: Chrome最新版、Safari最新版。
- インターネット接続、外部API、アカウント、サーバーを必要としない。
- 対戦再開データは端末内だけに保存し、外部送信しない。
- カード選択、実行ボタン、ライフ、フェーズ、不可能理由をPC・スマホの双方で読める。
- スマホの主要操作対象は、誤操作を避けられるタップ領域を確保する。
- カード表示と対戦計算は同じ状態データを参照し、表示専用の別計算を持たない。
- 画像が未読込でもゲーム状態を破壊せず、代替表示を出す。

## 9. ゲームルール

- 初期ライフ: 4000。
- 初手: 5枚。先攻の最初のドローはなし。
- 盤面: モンスター3枠、魔法・罠3枠。
- 通常召喚: 1ターンに1回。
- 上級モンスターは場のモンスターを素材にする。
- モンスターは攻撃表示・守備表示・裏側守備を使う。
- カード種類はモンスター・魔法・罠。
- 勝利条件は相手ライフを0以下にするか、相手がドローできない状態にすること。
- 目標試合時間は5〜10分。

## 10. 受け入れ基準

- [ ] 固定デッキを使い、CPUとの試合を開始から勝敗決定まで進行できる。
- [ ] モンスター・魔法・罠の20種類が定義され、双方の20枚デッキで使用される。
- [ ] 通常召喚、上級召喚、攻撃・守備・裏側守備、魔法、罠が機能する。
- [ ] 不可能な操作が無効化され、理由が表示される。
- [ ] 直前のターン開始時点から試合を再開できる。
- [ ] Chrome最新版とSafari最新版で基本進行を確認できる。
- [ ] PC横画面とスマホ縦画面で盤面・手札・操作UIが画面外へ欠落しない。
- [ ] 背景・カード絵・カード枠・アイコン・エフェクトが独立パーツとして管理される。
- [ ] PC・スマホの座標が中央レイアウト定義から算出され、コード中に個別座標が散在しない。
- [ ] 座標確認用デバッグ表示で、全スロットとエフェクト基準点を目視確認できる。
- [ ] 複数画面比率の配置テストが成功する。
- [ ] 承認済みモック `docs/mock/approved-20260717.png` と配色・質感・情報階層が一致する。
- [ ] 外部通信なしで動作し、対戦データが端末外へ送信されない。
